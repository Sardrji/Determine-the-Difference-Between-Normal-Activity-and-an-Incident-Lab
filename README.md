# Determine the Difference Between Normal Activity and an Incident

## Objective

The **Determine the Difference Between Normal Activity and an Incident** lab focused on investigating security findings generated by **Google Cloud Security Command Center (SCC) Event Threat Detection**. The primary objective was to analyze IAM-related threat findings, differentiate between legitimate administrative activity and malicious behavior, investigate audit logs in Cloud Logging, and remediate unauthorized privileged access. This hands-on exercise strengthened my understanding of cloud threat detection, incident investigation, and access control remediation.

### Skills Learned

- Understanding of Google Cloud Event Threat Detection.
- Experience investigating IAM security findings.
- Ability to distinguish legitimate activity from malicious behavior.
- Knowledge of Google Cloud Audit Logs and Cloud Logging.
- Experience analyzing privileged access events.
- Understanding of cloud incident investigation and remediation.
- Practical experience with Identity and Access Management (IAM) security.

### Tools Used

- Google Cloud Platform (GCP)
- Security Command Center (SCC)
- Event Threat Detection
- Google Cloud IAM
- Cloud Logging (Logs Explorer)
- Google Cloud Audit Logs

## Steps

### Step 1: Simulate Suspicious IAM Activity

Granted the **Project Owner** role to an external Gmail account to intentionally trigger an Event Threat Detection finding. This simulated an unauthorized privilege escalation scenario commonly associated with malicious account compromise.

---

### Step 2: Review Security Findings

Accessed **Security Command Center** and filtered Event Threat Detection findings related to **Persistence: IAM Anomalous Grant**. Reviewed the generated alerts to understand how Google Cloud detects unusual IAM permission changes.

---

### Step 3: Investigate the Findings

Examined the details of each IAM finding, including the principal that granted permissions and the recipient account. Compared legitimate administrative actions with suspicious privilege assignments to identify which finding represented expected activity and which indicated a genuine security incident.

---

### Step 4: Analyze Audit Logs

Used **Cloud Logging (Logs Explorer)** to inspect audit logs associated with IAM policy modifications. Reviewed authentication details, request metadata, source information, and authorization events to trace the origin of the privileged access request and validate the Security Command Center findings.

---

### Step 5: Remediate Unauthorized Access

Removed the Project Owner role from the unauthorized external account, eliminating the excessive privilege and restoring secure Identity and Access Management (IAM) configuration.

---

### Step 6: Complete the Lab

Successfully investigated cloud security findings, differentiated legitimate administrative activity from malicious behavior, analyzed audit logs, remediated unauthorized privileged access, and completed all required lab objectives.
